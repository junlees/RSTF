### 개선 여지
- **FPR 9.64% / 이상 검출률 90.36%**: 33,800개의 이상 샘플 중 3,259개 누락
  → NOK 샘플(6.72%)의 클래스 불균형 문제가 주요 원인
- `model_best.pth`가 save\_period 제약으로 실제 최적 에폭(13)이 저장되지 않음
  → `save_period` 값을 낮추거나 best 모델 즉시 저장 로직 수정 권장

### 클래스 불균형 대응 방안 (권고)
1. Loss에 **class weight** 적용 (`weight=[1.0, 13.86]` — 비율 역수)
2. **Focal Loss** 도입 (이상 샘플에 더 높은 손실 집중)
3. **ε 튜닝**: Rough Set ε를 높여 경계 영역을 확대, 이상 민감도 향상
4. **Oversample** NOK 또는 **Undersample** OK (SMOTE 등)
